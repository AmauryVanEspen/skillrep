SkillRep

An experiment in computing a skill focused reputation from StackOverflow data
About GitHub Discussions
This project was inspired by the very interesting StackRating but the approach here is different.
It 's mostly a filtering of the SO reputation to keep the gain that can most be related to some developer skill:
  • Question score is ignored
  • Only accepted answers are counted (might change, discussion welcome)
  • Self-answered questions are ignored
  • Closed questions are ignored
  • Score (upvotes minus downvotes) per answer is capped at 10
  • Skill Rep per answer is 10*score plus 15 when accepted
  • There's no daily reputation cap
  • Bounties are ignored
  • Votes coming more than one week after the question creation may be ignored

In the future I might also ponder the value of the votes (currently 10) using the skill rep the asker won (questions asked by good answerers seem to be better overall).

I would also have loved to ponder using the skill rep of the voters but unfortunately (for me) the API keep this data secret.
